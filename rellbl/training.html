Training¶

Model parameters and definitions¶

Each query tuple has the form \((rel, ent, dir)\), where:

  • rel is a relation (e.g. president-of.r),
  • ent is an entity (e.g. united-states.e),
  • dir is a direct (left or right).

For example, the tuple (president-of.r, united-states.e, right) would stand for the query ‘(president-of.r, ?, united-states.e)’.

Some key parameters (not necessarily explicitly needed):

  • \(d\): The embedding dimensionality.
  • \(D\): The number of rows in the dataset (or minibatch).
  • \(V_{rel}\): The size of the relation vocabulary.
  • \(C_{rel}\): The number of classes for the factorised relation vocabulary. We want \(C_{rel} \approx \sqrt{V_{rel}}\).
  • \(c_{rel}\): The maximum number of relations per class. We also want \(c_{rel} \approx \sqrt{V_{rel}}\).
  • \(V_{ent}\): The size of the entity vocabulary.
  • \(C_{ent}\): The number of classes for the factorised relation vocabulary. We want \(C_{ent} \approx \sqrt{V_{ent}}\).
  • \(c_{ent}\): The maximum number of relations per class. We also want \(c_{ent} \approx \sqrt{V_{ent}}\).

Data structures format¶

  • Training data (\(data\)): A \(D \times 3\) matrix, where each row \(data_i\) has as elements:

    • \(data_i[0]\): the word index of a relation (in its respective class),
    • \(data_i[1]\): the word index of an entity (in its respective class),
    • \(data_i[2]\): the index of the direction (0 for left, 1 for right).
  • Training data class (\(classes\)): A \(D \times 2\) matrix, where each row \(class_i\) has as elements:
    • \(classes_i[0]\): the class index of a relation,
    • \(classes_i[1]\): the class index of an entity.

  • Training betas (\(betas\)): A \(D \times d\) matrix, where each row \(betas_i\) is the embedding for a question aligned with the query tuple represented by \(data_i\) and \(classes_i\).

  • Relation vocab tensor (\(vocab_{rel}\)): A \(C_{rel} \times c_{rel} \times d\) tensor, where \(vocab_{rel}[i,j,:]\) is the embedding for the \(j\)-th relation in the \(i\)-th class. In other words, \(vocab_{rel}[i]\) is the \(c_{rel} \times d\) vocabulary matrix for the relations in class \(i\).

  • Entity vocab tensor (\(vocab_{ent}\)): A \(C_{ent} \times c_{ent} \times d\) tensor, where \(vocab_{ent}[i,j,:]\) is the embedding for the \(j\)-th entity in the \(i\)-th class. In other words, \(vocab_{ent}[i]\) is the \(c_{ent} \times d\) vocabulary matrix for the entities in class \(i\).

  • Relation unigram log-probability matrix (\(probs_{rel}\)): A \(C_{rel} \times c_{rel}\) matrix, where \(probs_{rel}[i,j]\) is the log_probability in ReVerb of the \(j\)-th relation of the \(i\)-th class.

  • Entity unigram log-probability matrix (\(probs_{ent}\)): A \(C_{ent} \times c_{ent}\) matrix, where \(probs_{ent}[i,j]\) is the log_probability in ReVerb of the \(j\)-th entity of the \(i\)-th class.

  • Relation class log-probabilities (\(classprobs_{rel}\)): A \(C_{rel}\)-dimensional vector, where \(C_{rel}[i]\) is the log-probability of the \(i\)-th relation class.

  • Entity class log-probabilities (\(classprobs_{ent}\)): A \(C_{ent}\)-dimensional vector, where \(C_{ent}[i]\) is the log-probability of the \(i\)-th entity class.

Classes and functions¶

rellbl.training.train_naive_factorized_generator(learning_rate, epochs, batch_size, train_data_in, train_classes_in, train_betas, rel_vocab, ent_vocab, rel_unigram_probs, ent_unigram_probs, rel_class_probs, ent_class_probs, save_folder, l2=0, shuffle=True)¶

Train a factorized relLBL with SGD.

Parameters:
  • learning_rate (float) – Learning rate for SGD.
  • epochs (int) – Number of epochs to train for.
  • batch_size (int) – Batch size for SGD minibatches.
  • train_data_in (numpy.ndarray) – Matrix (\(data\)) containing the training data.
  • train_classes_in (numpy.ndarray) – Matrix (\(classes\)) containing the training data classes.
  • train_betas (numpy.ndarray) – Matrix (\(betas\)) of training betas.
  • rel_vocab (numpy.ndarray) – Factorized vocabulary embedding tensor (\(vocab_{rel}\)) for relations.
  • ent_vocab (numpy.ndarray) – Factorized vocabulary embedding tensor (\(vocab_{ent}\)) for entities.
  • rel_unigram_probs (numpy.ndarray) – Matrix (\(probs_{rel}\)) of relation unigram log-probabilities (one row per class).
  • ent_unigram_probs (numpy.ndarray) – Matrix (\(probs_{ent}\)) of entity unigram log-probabilities (one row per class).
  • rel_class_probs (numpy.ndarray) – Vector (\(classprobs_{rel}\)) of relation class log-probabilities.
  • ent_class_probs (numpy.ndarray) – Vector (\(classprobs_{ent}\)) of entity class log-probabilities.
  • save_folder (string) – Path to folder where models should be saved.
  • l2 (float) – Lambda parameter for L2 regularisation.
  • shuffle (bool) – If True, shuffle data before start of training.
